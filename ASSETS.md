### Convention de nommage

`<YYYY-MM><LOC>-<camera-id>[-<process-tags>].jpg` — la convention FON d'origine est conservée pour traçabilité catalogue. Lowercase + tirets uniquement (URL-safe). Exemple :

```
2024-07egy-dive24-0r8a6273-2.jpg   ← capture 2024-07, Égypte, 24e plongée, ID caméra 0R8A6273
```

```
https://cdn.jsdelivr.net/gh/WMH-Project/neptune-assets@v1.0.9/photos/expeditions/<filename>
```

## Vérification d'une URL

```bash
# HEAD request pour vérifier la disponibilité + headers cache
curl -I 'https://cdn.jsdelivr.net/gh/WMH-Project/neptune-assets@v1.0.0/logos/mission-neptune/mission-neptune-white.png'
```

Headers attendus en production :
- `HTTP/2 200`
- `cache-control: public, max-age=604800, s-maxage=43200`
- `x-cache: HIT` (après le premier fetch)

## Purge du cache (rare, uniquement si override d'un asset existant)

```bash
curl https://purge.jsdelivr.net/gh/WMH-Project/neptune-assets@v1.0.0/<path>
```

> Préférer créer une nouvelle version (`v1.0.1`) plutôt que purger le cache d'un tag existant.
