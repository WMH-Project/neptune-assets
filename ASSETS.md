## Vérification d'une URL

```bash
# HEAD request pour vérifier la disponibilité + headers cache
curl -I 'https://cdn.jsdelivr.net/gh/WMH-Project/neptune-assets@v1.1.1/logos/mission-neptune/mission-neptune-white.png'
```

Headers attendus en production :
- `HTTP/2 200`
- `cache-control: public, max-age=604800, s-maxage=43200`
- `x-cache: HIT` (après le premier fetch)

## Purge du cache (rare, uniquement si override d'un asset existant)

```bash
curl -X POST https://purge.jsdelivr.net/ \
  -H 'Content-Type: application/json' \
  -d '{"path":["/gh/WMH-Project/neptune-assets@v1.1.1/<path>"]}'
```

> Préférer créer une nouvelle version (`v1.1.2`) plutôt que purger le cache d'un tag existant. La purge ne propage correctement que les fichiers ; les nouveaux tags peuvent prendre 1–2 minutes à apparaître dans le CDN après push.
